Chantier UX #2 · Mockup d'impact

Validation inline — l'erreur s'affiche sous le champ, pas après le submit

Aujourd'hui, l'utilisateur soumet, attend, voit une bulle navigateur ou un encart rouge en haut. Avec la validation inline, chaque champ se déclare invalide dès qu'il quitte le focus — message en rouge sous le label, bordure rouge, icône, et aria-invalid pour les lecteurs d'écran. Le wizard bloque « Suivant » tant que l'étape n'est pas valide et défile vers le premier champ en erreur.

02

Quand l'erreur apparaît · règle « touched »

Évite les faux positifs : on n'agresse pas l'utilisateur dès la première frappe. L'erreur apparaît au premier blur du champ, ou immédiatement sur tous les champs invalides à la première tentative de submit. Une fois en erreur, elle se met à jour à la frappe (pour disparaître dès que la valeur devient valide).
1 · ReposChamp vide. Aucune erreur affichée même s'il est requis.
→
2 · Focus + frappeL'utilisateur tape. Bordure verte. Pas d'erreur affichée.
→
3 · BlurQuitte le champ. Si invalide → erreur affichée. Si valide → check vert.
4 · Re-focus + frappeErreur déjà affichée → mise à jour temps réel.
Dès que la valeur devient valide → erreur disparaît.
↓
Submit malgré toutTous les champs invalides passent en erreur d'un coup.
Auto-scroll + focus sur le premier.

06

API attendue · FormField + useFieldValidation()

Extension a minima : FormField reçoit error et onBlur ; useFieldValidation isole la logique touched/error par champ. Aucune dépendance ajoutée.
// src/components/ui/FormField.tsx — props ajoutées interface FormFieldProps { id: string label: string value: string onChange: (v: string) => void onBlur?: () => void // nouveau error?: string | null // nouveau · si présent → état erreur required?: boolean ... } // src/hooks/useFieldValidation.ts — mini-hook par champ export function useFieldValidation<T>( initial: T, validate: (v: T) => string | null, ) { const [value, setValue] = useState(initial) const [touched, setTouched] = useState(false) const error = touched ? validate(value) : null return { value, error, onChange: setValue, onBlur: () => setTouched(true), forceShow: () => setTouched(true), // déclenché au submit isValid: validate(value) === null, } } // Usage typique const email = useFieldValidation('', (v) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(v) ? null : 'Format d\'adresse email invalide.' ) <FormField id="email" label="Email" required {...email} />
Garde-fou sécurité — la validation inline n'est jamais une garantie de validité. Toute donnée doit être re-validée côté serveur (Edge Function, contrainte DB, RLS). Les messages d'erreur affichés sont des strings statiques (pas d'interpolation depuis du contenu utilisateur non échappé) — pas de surface XSS.
