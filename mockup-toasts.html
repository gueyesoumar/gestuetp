Chantier UX #1 · Mockup d'impact

Système de toasts — feedback visuel sur chaque action

Aujourd'hui, sauvegarder un travail terrain, soumettre une validation ou uploader un document ne déclenche aucune confirmation. L'utilisateur doute. Avec un système de toasts cohérent (succès, erreur, info, chargement asynchrone), chaque action génère un retour immédiat, éphémère et accessible — sans alourdir l'écran.

03

Comparaison avant / après

Aujourd'hui, le bouton « Enregistrer » revient à son état initial sans confirmation visible. En cas d'erreur, un encart rouge persistant s'affiche en haut du formulaire. Les toasts remplacent ce pattern pour le feedback transient ; ErrorAlert reste utilisé uniquement pour les erreurs bloquantes (ex. chargement initial échoué).

✓ Message générique côté UI · détail loggé en console · disparait après 6 s · CTA contextuel
Garde-fou sécurité (CLAUDE.md §3) — le wrapper useToast() impose un message générique pour les erreurs. Le détail technique (message Supabase, stack) est loggé en console via console.error(), jamais affiché à l'utilisateur. Cette séparation empêche la fuite d'informations internes (noms de tables, contraintes, schémas) qui pourraient faciliter la reconnaissance d'attaque.
04

API attendue · useToast()

Wrapper minimal au-dessus de Sonner — impose la sémantique Gestu et masque les détails techniques en cas d'erreur.
// src/hooks/useToast.ts export function useToast() { return { success: (title: string, opts?: ToastOpts) => sonner.success(title, opts), error: (title: string, err?: unknown) => { if (err) console.error('[toast]', err) // log technique uniquement sonner.error(title) // message générique côté UI }, info: (title: string, opts?: ToastOpts) => sonner.info(title, opts), warn: (title: string, opts?: ToastOpts) => sonner.warning(title, opts), promise: <T>(p: Promise<T>, m: PromiseMessages) => sonner.promise(p, m), } } // Usage typique const toast = useToast() try { await saveControl() toast.success('Constat enregistré', { description: 'Prochaine étape : action corrective.' }) } catch (err) { toast.error('Impossible d\'enregistrer', err) // err loggé, pas affiché }
