2. Sous-ensemble Markdown autorise (option B)

Pour eviter une variabilite excessive entre auditeurs et limiter le risque XSS, on n'autorise que 4 elements. Pas de titres, pas de liens, pas d'images, pas de HTML brut.

4. Recommandation

Option B — Markdown limite Recommande

Pour ~2 jours de developpement supplementaire on gagne un rendu professionnel sur les 5 surfaces, une coherence inter-auditeurs et une lecture client largement plus claire. Le surcout est concentre sur le generateur PDF (le reste est mutualise).

  • Sous-ensemble strict : gras, italique, liste a puces, code inline. react-markdown + remark-gfm avec allowedElements verrouille.
  • Toolbar minimale dans le finding card editor — 4 boutons + 2 raccourcis (Ctrl+B, Ctrl+I).
  • Helper markdownToPlain() partage entre XLSX, notifications email et previews texte court.
  • Securite : sanitize cote rendu, jamais de raw HTML, conforme CLAUDE.md §3.
