Architecture proposée — Piste B

📦 Migrations DB

  • 1. Étendre plans avec max_users, max_missions, tier (free/standard/enterprise/custom).
  • 2. Créer plan_features (table de jonction) : plan_id + flag_id.
  • 3. Étendre feature_flags avec category (ai/reporting/branding/security/collab) et maturity (stable/beta/new).

🖥️ Pages admin

  • /admin/plans — Liste cards + Vue matrice + CRUD avec garde-fou suppression.
  • /admin/feature-flags (existant) — Garde la gestion globale ON/OFF + ajoute la category/maturity.
  • /admin/cabinets/:id onglet "Fonctionnalités" — refonte plan-based : "Inclus dans le plan", "Personnalisations", "Hors plan disponibles".

🔒 Logique RLS

  • Lecture publique sur plans + plan_features (déjà le cas pour plans).
  • Écriture platform_owner uniquement via Edge Function admin-plans (audit log).
  • Garde-fou DELETE : impossible de supprimer un plan si organizations.plan_id y pointe (ON DELETE RESTRICT).

⚙️ Edge Function admin-plans

Actions : create, update, delete, set_features (remplace la liste des features liées). Toutes loggées dans audit_log avec motif obligatoire.
